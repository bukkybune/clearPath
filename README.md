# ClearPath
### A Mobile Financial Decision Support Platform for College Students

ClearPath is a mobile application designed to help college students manage their personal finances through budgeting tools, investment and debt simulators, and interactive financial education.

---

## Live Demo

> Open in **Expo Go** on your phone — no setup required.

Run the following command after installing EAS CLI and logging in:
```bash
eas update --branch main
```
Or scan the QR code generated by `npx expo start` for local development.

---

## Project Description
Many college students lack the financial literacy and tools needed to make informed financial decisions. ClearPath addresses this gap by providing an accessible, student-focused platform with:
- Budget tracking with expense & income logging, category limits, and visual spending breakdowns
- Investment growth simulator using compound interest projections
- Debt payoff simulator for student loans and credit cards
- A financial learning hub with sequential lessons, quizzes, and guide articles
- Gamification — daily streaks, XP points, level progression, and milestone awards

## Setup Instructions

### Prerequisites
- Node.js (v18 or higher)
- Expo Go app on your mobile device

### Installation
```bash
# Clone the repository
git clone https://github.com/bukkybune/clearPath.git
cd clearPath

# Install dependencies
npm install

# Set up environment variables
cp .env.example .env
# Open .env and fill in your Firebase credentials
# (get these from Firebase Console → Project Settings → Your apps → Config)

# Start the development server
npx expo start
```

### Environment Variables
The app reads Firebase credentials from a `.env` file at the project root. See `.env.example` for the required keys:

```
EXPO_PUBLIC_FIREBASE_API_KEY=
EXPO_PUBLIC_FIREBASE_AUTH_DOMAIN=
EXPO_PUBLIC_FIREBASE_PROJECT_ID=
EXPO_PUBLIC_FIREBASE_STORAGE_BUCKET=
EXPO_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=
EXPO_PUBLIC_FIREBASE_APP_ID=
EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID=
```

> These variables are embedded into the app bundle at build time via Expo's `EXPO_PUBLIC_` prefix. Never commit your `.env` file.

### Running the App
1. Run `npx expo start` in the project directory
2. Scan the QR code with Expo Go on your phone
3. Both devices must be on the same WiFi network

### Publishing an OTA Update
```bash
eas update --branch main --message "description of changes"
```

- [x] Profile & Settings screen with editable display name and dark/light theme toggle
- [x] Empty states for all list and data views
